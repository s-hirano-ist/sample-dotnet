# ページングのLinkヘッダーを理解する

ページングされた一覧では、JSONの`page`や`totalPages`だけでなく、次に取得すべきURLをHTTPの`Link`ヘッダーで返せます。

```http
Link: </todos?page=1&pageSize=20>; rel="first", </todos?page=2&pageSize=20>; rel="next"
```

`rel`はURLの役割を表します。このAPIでは次の関係を使います。

- `first`: 最初のページ
- `self`: 現在のページ
- `prev`: 1つ前のページ。最初のページにはありません
- `next`: 1つ後のページ。最後のページにはありません
- `last`: 最後のページ。Todoが0件の場合にはありません

検索、完了状態フィルター、ソートを指定していた場合は、リンク先にも同じ条件を引き継ぎます。

## なぜヘッダーを使うのか

レスポンスJSONの形を変更せずに、クライアントが次のリクエスト先を利用できます。クライアントはURLを文字列連結で組み立てる必要がなくなり、サーバー側で定義したページングルールに従えます。

- ヘッダーとJSON本文のどちらにページ情報を置く設計がよいか

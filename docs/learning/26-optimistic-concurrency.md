# ETagで更新競合を防ぐ

ETagはキャッシュだけでなく、更新競合の検出にも使えます。

```text
1. クライアントAがGETしてETagを受け取る
2. クライアントBが先に更新する
3. クライアントAが古いETagでPUTする
4. APIが412 Precondition Failedを返す
```

PUTでは`If-Match`ヘッダーを使います。

```http
If-Match: "現在のETag"
```

ETagが一致した場合だけ更新・削除し、更新後は新しいETagを返します。
これを楽観的同時実行制御と呼びます。

## ステータスコード

```text
200 -> ETagが一致し、更新成功
204 -> ETagが一致し、削除成功
404 -> 対象Todoが存在しない
412 -> ETagが古く、別の更新が先に行われた
```

## 練習問題

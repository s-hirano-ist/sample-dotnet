# EF Coreでデータベースインデックスを追加する

インデックスは、検索や並び替えに使う列をあらかじめ整理したデータ構造です。検索時にテーブル全体を毎回読む量を減らせます。

このAPIでは一覧の使い方に合わせて、次のインデックスを追加しました。

- `IsDone, CreatedAt, Id`: 完了状態で絞り込み、作成日時で並べる検索
- `CreatedAt, Id`: 作成日時で並べる検索
- `Title, Id`: タイトル順で並べる検索

`Id`を末尾へ含めるのは、同じ値のときの安定した並び順を補助するためです。

## マイグレーション

インデックスもDBスキーマの一部なので、`TodoDbContext`だけを変更して終わりにせず、マイグレーションへ記録します。

```text
TodoApi/Migrations/20260719080505_AddTodoQueryIndexes.cs
```

アプリ起動時の`Database.Migrate()`によって、未適用のマイグレーションが順番に適用されます。

## 注意点

インデックスは追加すればよいわけではありません。読み取りは速くなりますが、INSERTやUPDATE時の維持コストとディスク使用量が増えます。タイトルの部分一致検索のように、通常のB-treeインデックスが効きにくい検索もあります。

本番では、実際のデータ量とSQLの実行計画を確認してから追加・削除を判断します。

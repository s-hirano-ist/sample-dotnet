# Redisで冪等性キーを共有する

インメモリ辞書はプロセスごとに別物です。コンテナAで保存した`Idempotency-Key`をコンテナBは知りません。Redisを共有ストアにすると、ロードバランサーの振り分け先が変わっても同じキーを参照できます。

```json
{
  "Idempotency": {
    "Store": "Redis",
    "EntryLifetimeSeconds": 300,
    "WaitTimeoutSeconds": 5
  }
}
```

## 処理中の予約

最初のリクエストは、RedisのLuaスクリプトでキーを`pending`として原子的に予約します。別コンテナが同じキーを受け取った場合は、同じリクエスト内容であることを確認して、作成処理の完了を待ちます。

作成が成功すると、`pending`の値をTodo付きの`completed`へ置き換えます。置き換え時にも予約者の値を確認するため、別の処理が自分の結果を上書きしないようにします。

## 障害時の考え方

- Redisが利用できない場合は、インメモリへ黙って切り替えない
- `pending`にはTTLを付け、プロセスが強制終了しても永久に残さない
- 待機時間を超えた場合は、作成成功と誤認させず`409 Conflict`を返す

RedisモードはComposeで有効にしています。単独起動やテストでは既定のインメモリモードを使います。

# APIキーの有効期限・失効・監査

APIキーは発行したら永久に使える状態にせず、クライアント単位でライフサイクルを管理します。

```json
{
  "Authentication": {
    "Clients": [
      {
        "Name": "frontend",
        "Key": "secret-from-user-secrets",
        "Permissions": ["todo:read"],
        "ExpiresAtUtc": "2026-12-31T00:00:00+00:00",
        "Revoked": false
      }
    ]
  }
}
```

- `ExpiresAtUtc`: 指定時刻以降は401を返す
- `Revoked`: `true`にすると期限前でも即時に無効化する
- `Name`: 監査ログでクライアントを識別する名前

## 監査ログ

認証成功時はクライアント名、パス、EventIdだけを記録します。APIキーの値は成功・失敗のどちらでもログへ出しません。

監査ログは、誰がいつどのAPIを使ったかを調査するために使います。個人情報や秘密情報をログへ追加しないことが重要です。

## 運用上の注意

- APIキー本体はUser SecretsやSecrets Managerで管理する
- 期限が近いキーを新しいキーへ切り替えるローテーション手順を用意する
- 失効操作を設定ファイルの変更だけに頼る場合は、反映時間を把握する
- 本番ではキー情報をDBや専用の認証基盤で管理する方式も検討する

# ETagと条件付きGETを理解する

`ETag`は、リソースの現在の内容を表す識別値です。
クライアントは次回のGETで`If-None-Match`を送り、内容が変わっていなければサーバーは`304 Not Modified`を返します。

```text
最初のGET
  <- 200 + ETag

次のGET
  -> If-None-Match: "..."
  <- 304（内容に変更なし）
```

304の場合、レスポンス本文を再送しないため通信量を減らせます。

このAPIではTodoのID、タイトル、完了状態、日時からSHA-256ダイジェストを作ってETagにします。
Todoが更新されるとダイジェストも変わるため、古いETagは一致しません。

## 練習問題

- `ETag`と`If-None-Match`はそれぞれどちら向きのヘッダーか
- 内容が変わっていないときに200ではなく304を返す利点は何か
- ETagへAPIキーや個人情報を含めてはいけない理由は何か

# カーソルページングを理解する

ページ番号方式の`Skip`は、後ろのページほど大量の行を読み飛ばす必要があります。カーソルページングでは、前回の最後の位置をカーソルとして渡し、その位置より後ろだけを取得します。

```http
GET /todos/cursor?pageSize=20

{
  "items": [],
  "pageSize": 20,
  "nextCursor": "eyJWZXJzaW9uIjoxLCJMYXN0VG9kb0lkIjoyMH0",
  "hasNextPage": true
}
```

次のリクエストでは、レスポンスの`nextCursor`をそのまま渡します。

```http
GET /todos/cursor?pageSize=20&cursor=...
```

このAPIのカーソルはID昇順を基準にしています。カーソルの内部形式はBase64URLで隠し、クライアントがIDを直接組み立てないようにしています。

## ページ番号方式との違い

- ページ番号方式: 任意のページへ移動しやすい。全件数を返しやすい
- カーソル方式: 大量データの連続取得に向く。途中のページへ直接移動しにくい

カーソル方式では、データ追加が途中にあっても、取得済みIDより後ろを読むためページの重複や取りこぼしを抑えられます。

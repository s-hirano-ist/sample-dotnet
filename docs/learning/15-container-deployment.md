# コンテナでAPIを動かす基本

Todo APIをECS on Fargateなどで動かす場合、アプリケーションのプロセスと実行環境の関係を理解する必要があります。

## 1. アプリケーションプロセス

`dotnet run`を実行すると、.NETのWebアプリケーションプロセスが起動します。

```text
dotnet TodoApi.dll
  ↓
ASP.NET Coreアプリケーション
  ↓
HTTPリクエストを待ち受ける
```

アプリケーションプロセスは、メモリ上に次のような状態を持ちます。

- DIコンテナ
- インメモリキャッシュ
- インメモリのレート制限カウンター
- ロガー
- DB接続

## 2. コンテナ

コンテナは、アプリケーションと実行に必要なファイル・設定をまとめた実行単位です。

```text
コンテナ
  ├── .NETランタイム
  ├── TodoApi.dll
  ├── 設定
  └── アプリケーションプロセス
```

コンテナを再起動すると、プロセス内のメモリは失われます。

そのため、重要なデータをプロセス内の変数だけに保存してはいけません。TodoはSQLiteへ保存しているため、アプリのメモリだけに保存するよりは再起動に強い構成です。

## 3. 複数コンテナ

負荷に応じてコンテナを複数起動することがあります。

```text
ロードバランサー
  ├── コンテナA
  ├── コンテナB
  └── コンテナC
```

各コンテナは別プロセスです。次の状態は自動的には共有されません。

- メモリ上の変数
- インメモリキャッシュ
- Memoryモードのレート制限カウンター
- ローカルファイル

複数コンテナで共有したいデータは、DB、Redis、オブジェクトストレージなど外部サービスへ保存します。

## 4. ロードバランサー

ロードバランサーは、受け取ったHTTPリクエストをコンテナへ振り分けます。

```text
クライアント
  ↓
ロードバランサー
  ├── コンテナAへ転送
  ├── コンテナBへ転送
  └── コンテナCへ転送
```

1回目と2回目のリクエストが、同じコンテナへ届くとは限りません。

そのため、ログのRequest IDやDBのデータなど、共有すべき情報を外部から確認できる設計が重要です。

## 5. ヘルスチェックとの関係

ロードバランサーは、コンテナへ定期的にヘルスチェックを送ります。

```text
ロードバランサー -> GET /health
```

`200`ならトラフィックを送り、`503`なら一時的に送らない判断ができます。

```text
コンテナ起動
  ↓
DBなどの接続確認
  ↓
正常ならロードバランサーへ参加
```

## 6. 設定値の渡し方

コンテナへ設定値を渡すときは、環境変数を使うことがよくあります。

```text
RateLimit__Store=Redis
ConnectionStrings__Redis=redis.example:6379
```

ASP.NET Coreでは、JSONの階層を`__`で表します。

```text
RateLimit:Store
  ↓
RateLimit__Store
```

APIキーなどの秘密情報は、通常のDockerイメージやGitへ含めず、シークレット管理機能から注入します。

## 7. ファイルDBの注意点

このプロジェクトは学習用としてSQLiteファイルを使っています。

```text
Data Source=todo.db
```

単一コンテナの学習環境では動かしやすい一方、複数コンテナでは注意が必要です。

- コンテナごとに別のファイルになる可能性がある
- コンテナ破棄でファイルが失われる可能性がある
- 複数コンテナから同じファイルを安全に共有しにくい

本番では、RDSなどの共有データベースを使う構成が一般的です。

## 8. レート制限との関係

Memoryモードでは、コンテナごとにカウンターが別になります。

```text
コンテナA: 10回まで
コンテナB: 10回まで
```

Redisモードでは、共有Redisのカウンターを使えます。

```text
コンテナA ─┐
コンテナB ─┼-> Redisの共有カウンター
コンテナC ─┘
```

複数コンテナで同じ制限を適用したい場合、共有ストアが必要です。

## 練習問題

コンテナAでTodoを作成した直後に、コンテナBへTodo一覧リクエストが届いた場合を考えてみてください。

確認するポイント:

- Todoデータがどこに保存されていれば見えるか
- インメモリ保存が不向きな理由
- SQLiteファイルを複数コンテナで使うときの注意点
- レート制限を共有するために何が必要か

# Idempotency-Keyで重複作成を防ぐ

ネットワーク障害では、クライアントがPOSTの結果を受け取れず、同じリクエストを再送することがあります。通常のPOSTは再送するたびに新しいTodoを作るため、重複が発生します。

`Idempotency-Key`を付けると、サーバーはキーとリクエスト内容を保存します。

```http
POST /todos
Idempotency-Key: create-todo-1

{"title":"Learn .NET"}
```

同じクライアントが同じキーと同じ内容で再送した場合、最初に作成したTodoを返します。2回目のレスポンスには`Idempotency-Replayed: true`を付けます。

同じキーで内容が異なる場合は、どちらのリクエストを採用すべきか判断できないため`409 Conflict`を返します。

## 現在の実装の制約

今回の保存先はSingletonのインメモリ辞書です。そのため、次の制約があります。

- APIプロセスを再起動するとキーの記録が消える
- 複数コンテナ間ではキーを共有できない
- 古いキーを自動削除する有効期限はまだない

本番環境ではRedisやデータベースへ保存し、有効期限と保存容量を設計します。
